# Doxen — Product Vision

## The Name

**Doxen** comes from:
- **"doxa"** (Greek) → knowledge, understanding, belief
- **"-en"** → modern system suffix (like "happen", "fallen")

> A system that produces, organizes, and serves knowledge

The dachshund connection is intentional. Like the breed that digs persistently to find what's hidden underground, Doxen traverses long codebases to extract implicit knowledge buried in logic and relationships.

---

## The Problem

Modern software has a **knowledge crisis**.

### Code is executable—but not understandable

- Logic is scattered across files, services, and abstractions
- Context lives in:
  - Pull request discussions
  - Slack threads
  - Tribal knowledge
- Documentation is:
  - Outdated
  - Incomplete
  - Not trustworthy

### AI made this worse (and better)

With LLMs and RAG systems, everyone wants "chat with your codebase." But the reality is:
- Poor retrieval quality
- Missing context
- Hallucinations

**Why?** Because codebases are not structured as knowledge systems.

### The Core Insight

> The problem isn't lack of documentation.
> It's lack of structured, verifiable, and retrievable knowledge.

---

## The Solution

### Doxen = Knowledge Layer for Code

Doxen transforms code into **structured, testable, and AI-ready knowledge**.

### What Doxen Does

#### 1. Deep Code Understanding

- Analyzes code across files and services
- Extracts:
  - Logic patterns
  - Relationships
  - Intent

#### 2. Structured Knowledge (not just text)

Generates:
- Specifications
- Schemas
- Flows
- Dependency graphs

Not just markdown—machine-usable structure.

#### 3. Testable Documentation

- Produces executable snippets
- Verifies:
  - Examples
  - Assumptions
  - Behaviors

Documentation you can trust because it's tested.

#### 4. RAG-Native by Design

Outputs optimized for:
- Vector embeddings
- Semantic retrieval
- Context injection

Not retrofitted for AI—built for it from day one.

#### 5. Multi-Modal Knowledge

Beyond text:
- Visual diagrams
- Flow representations
- Structured artifacts

But:

> **A compiler from code → knowledge**
